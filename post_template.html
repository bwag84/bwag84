Creating a Blog Post

To create a blog post, create a markdown file inside the src/content/blog/ directory.

Keep in mind that the file name will be used as the slug for the post. For example, if you name your file my-first-post.md, the slug will be my-first-post.

Starting from AstroPaper v1.4.0, you can also organize blog posts into subdirectories, making it easier to manage your content.

For example, if you want to group posts under web, you can place them in src/content/blog/web/. This site affects the post URL, so src/content/blog/web/astropaper-post.md will be available at /posts/web/astropaper-post/.

If you don't want subdirectories to affect the post URL, just prefix the folder name with an underscore _.

Frontmatter

Frontmatter is the main place to store some important information about the blog post (article). Frontmatter lies at the top of the article and is written in YAML format. Read more about frontmatter and its usage in astro documentation.

Here is the list of frontmatter property for each post.

Property Description Remark
title Title of the post. (h1) required*
description Description of the post. Used in post excerpt and site description of the post. required*
pubDatetime Published datetime in ISO 8601 format. required*
modDatetime Modified datetime in ISO 8601 format. (only add this property when a blog post is modified) optional
author Author of the post. default = SITE.author
slug Slug for the post. This field is optional but cannot be an empty string. (slug: "") default = slugified file name

Tip! You can get ISO 8601 datetime by running new Date().toISOString() in the console. Make sure you remove quotes though.

Adding table of contents

By default, a post (article) does not include any table of contents (TOC). To include TOC, you have to specify it in a specific way.

Write Table of contents in h2 format (## in markdown) and place it where you want it to be appeared on the post.

For instance, if you want to place your table of contents just under the intro paragraph (like I usually do), you can do it like this.

---
# some frontmatter
---

Headings

There's one thing to note about headings. The AstroPaper blog posts use title as the main heading 1. Therefore, the rest of the heading in the post should be heading 2 or lower. In other words, in the markdown file, we should use ## and lower.

Storing Images for Blog Content

Here are two methods for storing images and other media for blog content:

Inside src/assets/ directory (recommended)

You can store images inside src/assets/ directory. For example, if the image is stored at src/assets/images/example.jpg, you can use it like this:

![something](../../assets/images/example.jpg)

Inside public directory

You can also store images inside the public directory. For example, if the image is stored at public/assets/example.jpg, you can use it like this:

![something](/assets/example.jpg)
